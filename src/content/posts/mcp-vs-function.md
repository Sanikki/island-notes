> **Function Calling是Agent的“嘴”（说出意图），MCP是Agent的“筋脉”（传输动作）。嘴决定要打哪招，筋脉把内力（请求）准确送到对应的指尖（工具）。**

Function Calling是Agent与LLM之间的通信协议，目的是让大模型回答的符合一定格式，方便程序解析

MCP是Agent与Tool之间的协议，约定怎么传参，怎么接收返回值

# Function Calling（函数调用）：大侠的“出招念头”

这是**大模型API自带的一种能力**（OpenAI、Anthropic、Gemini都支持）。它不是代码，而是**模型输出的一种特殊格式**。

**白话解释**：你问Agent“沈阳天气怎么样？”。Agent脑子里（大模型）飞速思考后，**不直接输出“晴转多云”**，而是吐出一段**JSON格式的“念头”**，比如：

```json
{
  "name": "get_weather",
  "arguments": {"city": "Shenyang"}
}
```

- **这个念头的含义**：“我决定调用‘查天气’这个函数，参数是沈阳。”
- **谁产生的**：大模型**自己推理出来的**（它是模型训练时学会的技能）。
- **谁执行的**：你（开发者）写的代码，在后台捕捉到这个JSON，去真的执行`get_weather`函数，拿到结果再喂回给模型。

**本质**：Function Calling是一种**“输出格式约束”**，让大模型学会“用工具的名字和参数”来回答问题，而不是直接说人话。

# MCP（模型上下文协议）：大侠的“兵器库统一接口”

这是**工具（Server）和Agent（Client）之间的通信标准**。

- **白话解释**：你不用管“查天气”这工具是用Python写的还是Node.js写的，也不用管它部署在本地还是云端。你只要按照MCP的规矩（JSON-RPC格式）发一条“请求执行get_weather”的消息，MCP Server就会帮你执行并把结果送回来。
- **这个通道的作用**：标准化**“调用请求的格式”**和**“传输方式”**（本地stdio或远程HTTP）。

**本质**：MCP是一种**“连接器规范”**，让工具的实现细节对Agent完全透明。

# 核心区别

| 维度           | Function Calling                           | MCP                                   |
| -------------- | ------------------------------------------ | ------------------------------------- |
| **处于哪一层** | **决策层**（大脑的“想法”）                 | **执行层**（手脚的“动作”）            |
| **产生方**     | 大模型本身（API返回的JSON）                | 开发者/平台制定的通信协议             |
| **核心问题**   | “我（Agent）**应该**调用哪个工具？”        | “我（Agent）**怎么**去调用这个工具？” |
| **依赖关系**   | 强依赖具体**大模型厂商**（如OpenAI的格式） | **不依赖**具体模型，任何Agent都能用   |
| **灵活性**     | 模型可能“瞎编”不存在的工具名（幻觉）       | 通过协议强制发现和验证工具是否存在    |

# 现实中的“黄金搭档”（它们怎么配合）

在真正的生产代码里，它们是**连在一起用的**：

**第一步（Function Calling产生念头）**：
用户问：“沈阳天气如何？”
Agent向大模型发起请求，并附上工具列表`[{"name":"get_weather", ...}]`。
大模型（OpenAI）返回：

```json
{"name": "get_weather", "arguments": {"city": "沈阳"}}
```



**第二步（MCP负责执行）**：
你的代码拿到这个JSON后，**不再写死**`if name == "get_weather": call_api()`**，而是会发一个MCP请求（JSON-RPC格式）给Server，Server收到后，执行对应代码，最后把结果返回**：

```python
mcp_client.call_tool(tool_name="get_weather", args={"city": "沈阳"})
```

**第三步（喂回模型）**：
把天气结果塞回给大模型，让它组织语言回复用户。
