> **江湖传言：得 MCP 者，可号令天下智能体，统三千工具，破万法壁垒。欲练此功，不必自宫，但需通晓「三层架构、三大法器、两道密道」。**

## 一、江湖困局：为何群雄并起却难成大事？

昔年江湖，各大门派各自为政：

- **五岳剑派（OpenAI）** 有自家的「Function Calling 剑诀」
- **逍遥派（Anthropic）** 有独门的「Tool Use 心法」
- **丐帮（Google）** 使的是「Function Declarations 打狗棒法」

一位大侠（智能体）想同时调用「嵩山派的查天气 API」、「丐帮的数据库」、「武当的 Slack 飞鸽传书」，**得先学会三家的暗号、拜帖格式、入门规矩**——累死大侠，折腾死开发者。

**MCP 的出现，如同武林中颁下《武林盟主令》：**

**从此天下英雄，统一拜帖格式，统一传讯暗号。无论你是哪门哪派的智能体，只要手持 MCP 令牌，便能号令所有归顺的 MCP Server，即插即用，无需适配。**

## 二、三层架构：武林总舵 · 传令兵 · 分舵

```plain
┌─────────────────────────────────────────────────────────────┐
│              总舵（Host）—— 武林盟主坐镇处                   │
│          如：Claude Desktop、你要做的AI应用              │
│               ↓ 内置                                      │
│               传令兵（MCP Client）                          │
│           负责传递盟主号令，沟通各大分舵                      │
│               ↓ 持盟主令（统一协议）                        │
│    ┌──────────┼──────────┬──────────┐                     │
│    ↓          ↓          ↓          ↓                     │
│  📂分舵     🗄️分舵     💻分舵      📨分舵                  │
│  (文件系统)  (数据库)   (GitHub)   (Slack)                 │
│  各分舵只做一件事：收到令旗，执行任务                         │
└─────────────────────────────────────────────────────────────┘
```

**三层职责，如同武林分工：**

| 层级                 | 武林比喻             | 职责                                         |
| -------------------- | -------------------- | -------------------------------------------- |
| **Host（总舵）**     | 武林盟主（如张三丰） | 发号施令，与群雄对话，不亲自动手             |
| **Client（传令兵）** | 武当知客道人         | 持盟主令跑遍各分舵，传话、接信               |
| **Server（分舵）**   | 各派执事弟子         | 只会一门绝活：读文件就只读文件，查库就只查库 |

**一句话：** 总舵动嘴，传令兵跑腿，分舵动手。

## 三、三大法器：Tool · Resource · Prompt

盟主令牌之下，MCP 赐予智能体三大神器：

| 法器                   | 本质          | 武林比喻           | 江湖实例                                                   |
| ---------------------- | ------------- | ------------------ | ---------------------------------------------------------- |
| **Tool（兵器）**       | 动词·执行动作 | **大侠的手脚**     | `出刀(目标)`、`飞鸽传书(内容)`、`点穴(穴位)`               |
| **Resource（藏经阁）** | 名词·提供数据 | **大侠的眼睛**     | `藏经阁/少林/易筋经`、`卷宗室/户部/人口册`                 |
| **Prompt（锦囊）**     | 指导性模板    | **大侠的锦囊妙计** | 「按六脉神剑的套路审这段代码」「用韦小宝的语气回这封邮件」 |

### 🗡️ Tool 与 📜 Resource 的生死之别

**Tool 是「打出去的一掌」——改变世界；Resource 是「看出去的一眼」——读取世界。**

| 维度         | Tool（兵器）                           | Resource（藏经阁）               |
| ------------ | -------------------------------------- | -------------------------------- |
| 江湖类比     | 大侠**出招**（有杀敌之效）             | 大侠**翻阅秘籍**（只读不写）     |
| 对江湖的影响 | **改变外部**：杀了人、送了信、删了文件 | **不改变任何事**：仅仅是看了本书 |
| 典型命例     | `飞鸽传书(收件人, 内容)`               | `藏经阁/少林/金刚经`             |
| 大侠心法     | **「我要做点什么」**                   | **「我得看点资料」**             |

**实例区分：**

- Tool：`射雕英雄传/出箭(目标: 大雕)` → 雕被射下来了（改变了江湖）
- Resource：`射雕英雄传/秘籍/九阴真经` → 大侠打开看了看内容（啥也没变，只是获得了知识）

## 四、传讯流程：一道盟主令的江湖之旅

**完整流程：** `大侠开口 → 盟主动脑 → 传令兵持令出寨 → 分舵执事动手 → 消息送回 → 大侠回复`

```plain
┌─────────────────────────────────────────────────────────────────┐
│ ① 大侠问话：「今日沈阳天气如何？」                            │
├─────────────────────────────────────────────────────────────────┤
│ ② 总舵（LLM）拆解任务：「此事需调用『天机阁』的查天气法器」   │
├─────────────────────────────────────────────────────────────────┤
│ ③ 传令兵（Client）持令出寨，通报各分舵：                      │
│    「天机阁听令！即刻查沈阳天气，速速回报！」                  │
├─────────────────────────────────────────────────────────────────┤
│ ④ 天机阁分舵（MCP Server）执行真功夫：                        │
│    弟子爬上天台观星象、翻黄历、调天眼API...                    │
│    回令：「沈阳今日晴，35°C，南风三级」                       │
├─────────────────────────────────────────────────────────────────┤
│ ⑤ 总舵收到回令，大侠组织语言：                                │
│    「少侠，今日沈阳热得紧，35°C，记得打伞。」                  │
└─────────────────────────────────────────────────────────────────┘
```

## 五、密道双途：盟主令如何传遍江湖？（传输方式）

MCP 令牌可通过两条密道传递：

| 密道                | 全称          | 适用场景                         | 江湖比喻                                                     |
| ------------------- | ------------- | -------------------------------- | ------------------------------------------------------------ |
| **stdio**           | 标准输入/输出 | **本地发令**，总舵与分舵同处一室 | **当面传音**：总舵主喊一嗓子，分舵弟子直接听到，最快、最稳、最安全 |
| **Streamable HTTP** | 可流式 HTTP   | **远程发令**，分舵远在千里之外   | **飞鸽传书**：令牌写在信上，飞鸽跨山越岭送往远方分舵         |

**新版密道（2025-03-26 修订）：**

- ❌ 旧版：分两路传讯（一路送信、一路收信），搞两个信鸽
- ✅ 新版：**一路直通**！一只信鸽既能送信又能收信，简化了传讯

### 🧠 特殊密道：内存传输（In-Memory Transport）

专为**闭关修炼**（开发测试）设计的密道：

不经过任何真实江湖通道，**直接在总舵主脑子里传递念头**。

```typescript
// 相当于总舵主闭眼冥想，消息在脑子里转了一圈
const [左路, 右路] = InMemoryTransport.createLinkedPair();
// 总舵主传令给自己，自问自答，绝不外泄
```

**用途：** 练功试招（单元测试）、验证令旗好不好使，不出山门就能完成。

## 六、MCP vs Function Calling：剑法与令旗之别

**Function Calling 是侠客「脑子里的出招念头」，MCP 是「江湖统一的令旗格式」。**

| 维度     | Function Calling（剑法）                                     | MCP（盟主令）                                     |
| -------- | ------------------------------------------------------------ | ------------------------------------------------- |
| 本质     | 大侠脑子里想的 **「我该使出什么剑法」**                      | 大侠实际持在手中的 **「令牌长什么样、该怎么传」** |
| 谁产生   | 大侠的内功心法（大模型 API 自己推理出来的 JSON）             | 武林盟指定的铁律（MCP 协议规范）                  |
| 核心问题 | 「我**应该**拔剑还是拔刀？」                                 | 「我**怎么**让传令兵把剑递到我手上？」            |
| 依赖     | 依赖大侠修炼哪家门派（OpenAI 的 JSON 格式 vs Anthropic 的不同） | **不依赖门派**！只要手持 MCP 令牌，任何侠客都能用 |
| 风险     | 大侠可能走火入魔，瞎编一套不存在的剑法                       | 令牌事先在兵器谱上登记过，不存在「瞎编」          |

### 组合出招（实战口诀）

```plain
大侠（用户）问：沈阳天气如何？
    ↓
① 剑法出鞘（Function Calling）
    大侠脑中运转（Agent）：「此问需召『天机阁』的查天气剑法」
    口吐口诀（统一格式）：{"剑名":"get_weather", "内劲":{"city":"沈阳"}}
    ↓
② 令旗传讯（MCP）
    传令兵手持刻有该口诀的盟主令（MCP统一格式），快马奔向天机阁（MCP）
    天机阁弟子见到令牌，依令执行真正功夫（服务）
    ↓
③ 回报大侠
    天机阁回令（统一返回格式），大侠组织语言回复
```

**Function Calling 管的是「大侠想叫什么剑法」，MCP 管的是「令旗怎么传到对应分舵」。**

## 七、动态点将：为何不再写死「if-else」？

**昔年笨办法（点将点名册写死）：**

```python
if 将军名 == "郭靖":
    郭靖.守襄阳()
elif 将军名 == "黄蓉":
    黄蓉.打狗棒法()
elif 将军名 == "杨过":
    杨过.黯然销魂掌()
# 每新来一位将军，就要往点名册上加一行 —— 累死兵部侍郎
```

**MCP 的「动态点将」：**

```python
# 1. 盟主大旗一挥：各分舵上报你的将领名单！
名册 = mcp_client.list_tools()
# 回报：["郭靖", "黄蓉", "杨过", "令狐冲", ...]  —— 实时获取
# 2. 大侠看完名册，选择叫谁（大模型 Function Calling 自动选）
# 3. 传令兵持令去找对应将军（MCP 动态调用）
result = mcp_client.call_tool(将军名, **内劲参数)
```

**精髓：** 新来一位「张无忌」，只需新开一个 MCP Server 分舵，**大侠的调令代码一行不改**。分舵重启，名册自动刷新。

## 八、自创武功：我自己写的 Tool 能否入盟？

**当然可以！** 少林七十二绝技都是历代高僧自创的，只要向盟主报备（包装成 MCP Server），便可入盟。

**三种入盟段位：**

| 段位             | 做法                                | 江湖场景                               |
| ---------------- | ----------------------------------- | -------------------------------------- |
| **⚔️** **野路子** | 装饰器 + 全局名册（自己搞个兵器谱） | 独行侠自己用，不与他人共享             |
| **📜** **正规军** | 本地 MCP Server（stdio）            | 加入总舵（Claude Desktop），供盟主调遣 |
| **🚀** **宗师级** | 远程 HTTP MCP Server                | 开宗立派，本门绝学供天下调用           |

```python
# 简单示例：野路子入盟，刻入自家兵器谱
兵器谱 = {}

def 刻入兵器谱(招式):
    兵器谱[招式.__name__] = 招式
    return 招式

@刻入兵器谱
def 天机阁_查天气(城市: str):
    return f"{城市}：烈日当空，35°C"

# 大侠开机自动加载兵器谱
print(兵器谱.keys())  # ['天机阁_查天气']
result = 兵器谱["天机阁_查天气"](城市="沈阳")
```

## 九、三大盟令之争：MCP · A2A · ANP

江湖之上，盟令不止一道，各有各的辖域：

| 盟令    | 管辖范围                                 | 类比                            |
| ------- | ---------------------------------------- | ------------------------------- |
| **MCP** | 大侠 **调用兵器/秘籍** 的接口标准        | **USB-C 统一接口**（手）        |
| **A2A** | 大侠与 **另一位大侠** 如何对话协作       | **江湖黑话/传音入密**（嘴）     |
| **ANP** | 如何在茫茫江湖中 **找到并信任** 某位大侠 | **身份证 + 江湖通讯录**（身份） |

**一句话：** MCP 管「手」（干活），A2A 管「嘴」（聊天），ANP 管「身份」（找人与防伪）。
